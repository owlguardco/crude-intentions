# SECURITY_AUDIT_2026-04-30.md

Scope: every `src/app/api/**/route.ts` not in the prior audit (analyze-setup, webhook-signal, journal/route.ts). 28 routes reviewed.

| ID    | Severity | File                                                       | Finding |
|-------|----------|------------------------------------------------------------|---------|
| **F-13** | **HIGH** | `src/app/api/journal/[id]/outcome/route.ts` | **PATCH has no auth check at all.** Any unauthenticated caller can close any journal entry with an arbitrary `close_price` (recalculates calibration, fires post-mortem, mutates market memory) or overwrite the `postmortem` field with arbitrary text via the `PostmortemPatchSchema` branch. Both client (`journal/page.tsx`) and `lib/journal/post-mortem.ts` send `x-api-key`, but the route never validates it. **Honest mistake**, not intentional — the sibling `outcome/postmortem/route.ts` IS auth-gated, and `firePostMortem(req,id)` in this file forwards `INTERNAL_API_KEY` as if it expected it to be checked. |
| **F-14** | **HIGH** | `src/app/api/eia-analysis/route.ts` | **No auth + no rate limit + unsanitized prompt injection + paid Anthropic call.** Body fields (`actual`, `expected`, `cushing`, `gasoline`, `distillates`, `refinery`) are interpolated directly into the user prompt without Zod validation. Any unauthenticated caller can: (a) drain Anthropic credits, (b) prompt-inject ALFRED via the inventory fields, (c) trigger arbitrary Anthropic calls to a model. Also uses **stale model `claude-sonnet-4-20250514`** instead of project-standard `claude-sonnet-4-5`, leaks `err.message` in the catch (line 55), and `JSON.parse(clean)` will throw on bad model output and bubble that message back. **Honest mistake**, this looks like an early prototype that never got auth bolted on. |
| **F-15** | **MED** | `src/app/api/cl-price/route.ts` | **No KV cache + no rate limit + no auth.** Every request hits Yahoo. A single attacker can fan out unbounded GETs to `query1.finance.yahoo.com/v8/finance/chart/CL=F` from this endpoint, getting the deployment IP-banned by Yahoo. Comparable routes (`ovx`, `geo-flag`, `street-pulse`) cache for 5–10 min; `cl-price` does not. Additionally line 124 leaks `err.message` in 502 responses. **Honest mistake** — pattern departs from sibling routes. |
| **F-16** | **MED** | `src/app/api/street-pulse/route.ts` | **No rate limit + no auth + 4-feed fanout on cache miss.** GET fans out to Yahoo (×2), Reuters, Investing.com in parallel. KV cache (10 min) limits steady-state damage, but cache poisoning isn't possible here so an attacker can simply force re-fetches with timing or after the cache TTL. Could get the deployment IP-banned by any of the four providers. **Intentional design** (dashboard widget, no auth by spec) but missing a rate-limit floor. |
| **F-17** | **MED** | All 5 `src/app/api/market-context/**/route.ts` files | **`detail: String(err)` leaks raw error text in 500 responses.** Files: `market-context/route.ts:84`, `market-context/reset/route.ts:47`, `market-context/fvg/route.ts:127`, `market-context/fvg/[id]/route.ts:82,113`, `market-context/idea/route.ts:124`, `market-context/idea/[id]/route.ts:87,118`. Same class as F-7 from the prior audit, but in routes that ARE auth-gated — so impact is limited to authenticated callers seeing internal error strings. **Honest mistake** — repeated copy-paste pattern. |
| **F-18** | **MED** | `src/app/api/eia-analysis/route.ts` | **Stale model `claude-sonnet-4-20250514`.** Project convention (and durable feedback memory) is `claude-sonnet-4-5`. Counted separately from F-14 because even after fixing auth/sanitization this would still need a rename. **Honest mistake** — predates the model-pinning convention. |
| **F-19** | **LOW** | All 18 routes using `token === INTERNAL_API_KEY` (or `auth === INTERNAL_API_KEY`) | **Auth comparison is not constant-time.** F-12 from the prior audit closed three routes (analyze-setup, webhook-signal, journal/route.ts) by routing through `safeEq()`. Eighteen other auth-gated routes still use plain `===`: `calibration`, `cron/weekly-brief` (CRON_SECRET via `===`), `cron/weekly-brief/trigger`, `fvg-scan`, `fvg-scan-auto`, `journal/[id]/postmortem`, `journal/backtest`, `journal/import`, `journal/observer`, `market-context` (×6 files), `mtf-consensus`, `position`, `supply-context`. **Honest mistake** — each route copy-pasted the legacy isAuthorised pattern; the F-12 fix landed in only three places. |
| **F-20** | **LOW** | `src/app/api/journal/import/route.ts` | **`err.message` leaked in per-trade `errors[].message`** at lines 100 and 151 — surfaces raw write/close error text back to the caller for each failed trade. Auth-gated so impact is bounded, but pattern matches F-7. **Honest mistake**. |
| **F-21** | **LOW** | `src/app/api/journal/postmortem-health/route.ts` | **Unauthenticated info disclosure** of which env vars are set (`VERCEL_APP_URL`, `INTERNAL_API_KEY`, `ANTHROPIC_API_KEY`). Returns booleans only, never values. The route doc explicitly says "stays safe to expose unauthenticated" — **intentional design choice**, but worth flagging because it tells an external attacker which auth chain to target. Could be moved behind `safeEq` against `INTERNAL_API_KEY` for trivial hardening with no cost to the curl-based health check (Railway can stamp the key). |
| **F-22** | **LOW** | `src/app/api/journal/backtest/route.ts` | **Unbounded loop driven by KV state.** When `signal_ids` is omitted, the route iterates EVERY OPEN backtestable journal entry, sleeping 500 ms between Yahoo fetches. With 600 OPEN entries, the loop runs ~5 minutes — close to Vercel's 300 s function ceiling. Auth-gated, so the attacker is the trader, but a runaway request can still wedge calibration writes mid-loop. **Honest mistake** — missing a hard cap on `eligible.length`. |
| **F-23** | **LOW** | `src/app/api/webhook/route.ts`, `src/app/api/webhook-close/route.ts` | **Webhook secret accepted via `?secret=` query string** (in addition to header/HMAC). Query strings end up in CDN access logs, Vercel deployment logs, browser/proxy history. **Intentional design choice** — TradingView alert URLs are the only practical fit and TradingView can't reliably set custom headers — but the secret should ideally be rotated more frequently than `INTERNAL_API_KEY` for that reason. Worth a doc note in `SECURITY.md`, not a code fix. **Operational note: WEBHOOK_SECRET should be rotated more frequently than INTERNAL_API_KEY — recommended every 30 days or on each Railway redeploy — because it appears in TradingView alert URLs which are logged by CDN and browser history.** |
| **F-24** | **LOW** | `src/app/api/cron/weekly-brief/route.ts` | **Authorization chain uses `===` not constant-time.** Lines 81 and 83 compare `auth.slice(7) === CRON_SECRET` and `direct === CRON_SECRET`. F-19-class but called out separately because this route uses `CRON_SECRET` rather than `INTERNAL_API_KEY` — would need its own constant-time wrapper or to pass `CRON_SECRET` through the existing `safeEq()`. |

**Summary**

- **HIGH**: 2 (F-13 outcome PATCH no auth, F-14 eia-analysis no auth + injection + paid LLM)
- **MED**: 4 (F-15, F-16, F-17, F-18)
- **LOW**: 6 (F-19, F-20, F-21, F-22, F-23, F-24)

**Recommended priority for fixes:**
1. **F-13 first** — write path with no auth, mutates calibration. This is the most exposed surface in the codebase.
2. **F-14 second** — paid LLM call with no auth.
3. **F-19 + F-24 together** — single constant-time wrapper, mechanical replacement across 19 files.
4. **F-15 / F-16** — add KV cache to `cl-price`, add rate-limit floor to `street-pulse`.
5. **F-17 / F-20** — drop `detail:` strings (mechanical).
6. **F-18** — rename model string.
7. **F-21 / F-22 / F-23** — doc updates / minor hardening.
